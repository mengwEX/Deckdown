# Deckdown Specification Index

Deckdown's specification is split by responsibility:

- [Architecture](./ARCHITECTURE.md): product boundaries, runtime architecture, package boundaries, data model, security model, and testing strategy.
- [Tech Stack](./TECH_STACK.md): final framework and dependency decisions.
- [Format](./FORMAT.md): `.dd` V0.1 file format and AI authoring contract.
- [Roadmap](./ROADMAP.md): implementation phases from architecture lock to desktop product.

The current root-level browser implementation is a spike. The production codebase should move toward the monorepo architecture described in `ARCHITECTURE.md`.
